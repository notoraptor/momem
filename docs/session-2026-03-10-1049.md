## 2. Revue du README (devenu DESIGN.md)

Analyse complète du README qui servait de cahier des charges. 7 points soulevés, tous résolus :

1. **Chemins absolus** : si le chemin source est absolu, `momem memorize` exige un second argument (chemin relatif dans le codebase).
2. **Convention `PROJECT_DIR/PROJECT_DIR/momem`** : ajout d'une config globale `default_project_dir` avec hiérarchie de résolution à 3 niveaux (config locale > config globale > convention par défaut).
3. **`momem show`** : ajout de `momem show --local` pour afficher les snippets installés dans le projet courant.
4. **Gestion des conflits** : en cas de conflit, momem refuse l'opération sauf avec `--force`.
5. **Dépendances entre snippets** : calculées à la volée via `ast.parse` (imports `momem.*`), résolution récursive avec détection de cycles.
6. **Portée Python-only** : confirmée, cohérente avec le parsing AST.
7. **Coquilles** : corrigées dans le README.

Le README a été renommé en `DESIGN.md` (cahier des charges) et un nouveau `README.md` minimal a été créé.

## 3. Évaluation de l'utilité du projet (RATIONALE.md)

Discussion sur les alternatives (pip install -e, pip install git+url, git submodules/subtree, vendoring). Conclusion : momem occupe une niche entre le copier-coller et le package structuré, avec l'avantage spécifique de gérer des snippets éparpillés. Avis sauvegardé dans `RATIONALE.md`.

## 8. Résolution du problème de namespace

### Imports relatifs
- Problème : les imports `from momem.xxx` dans les snippets ne correspondent pas au chemin d'installation (`PROJECT/PROJECT/momem/xxx`).
- Solution : réécriture automatique des imports en relatifs au `memorize` (`from momem.helper import x` → `from .helper import x`), avec calcul du nombre de dots selon la profondeur du fichier.
- `find_momem_imports` mis à jour pour détecter à la fois les imports absolus (legacy) et relatifs.

### Vérification des dépendants au forget
- `momem forget` vérifie maintenant si d'autres snippets dépendent du fichier. Refus sauf avec `--force`.
- Nouvelle fonction `find_dependents()` pour le scan inverse.

### Renommage du package interne
- Problème : conflit potentiel entre le package `momem` (outil CLI) et le dossier `momem/` (snippets) dans les projets.
- Solution : renommage du package Python interne de `momem/` en `momemcli/`.
- La commande CLI reste `momem`, le nom PyPI reste `momem`.
- Configuration hatch ajoutée : `packages = ["momemcli"]`.
